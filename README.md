# agent-dvr-bypass
Удаленный доступ для Agent DVR

# Инструкция по настройке бесплатного удаленного доступа к Agent DVR

Данное руководство описывает, как настроить обход блокировки внешних IP-адресов в Agent DVR без покупки платной подписки, используя связку веб-сервера **Nginx** (в качестве прозрачного прокси) и роутера **Keenetic**.

---

## Шаг 1. Настройка роутера Keenetic

Необходимо пробросить порты для веб-интерфейса и WebSockets-видеопотока.

1. Откройте веб-интерфейс Keenetic и перейдите в меню **Сетевые правила** -> **Переадресация портов**.
2. Убедитесь, что серверу с Ubuntu назначен **статический локальный IP** (в данном примере `192.168.1.149`).
3. Добавьте следующие правила и обязательно **включите** их (ползунки должны гореть синим):

| Вход (Интерфейс) | Протокол | Порт интерфейса | IP клиента | Порт на клиенте | Описание |
| :--- | :--- | :--- | :--- | :--- | :--- |
| Ваше подключение (CONVEX) | TCP | `8888` | `192.168.1.149` | `8888` | Nginx Proxy Web |
| Ваше подключение (CONVEX) | TCP/UDP | `3478` | `192.168.1.149` | `3478` | Agent DVR STUN |
| Ваше подключение (CONVEX) | UDP | `50000-50100` | `192.168.1.149` | `50000-50100` | Agent DVR WebRTC |
| Ваше подключение (CONVEX) | TCP | `22` | `192.168.1.149` | `22` | SSH (опционально) |

---

## Шаг 2. Редактирование конфигурации Agent DVR

Чтобы Agent DVR доверял нашему прокси-серверу и не блокировал трафик, нужно изменить параметры в конфигурационном JSON-файле.

1. Подключитесь к Ubuntu по SSH и откройте файл конфигурации:
   ```bash
   sudo nano /opt/AgentDVR/Media/XML/config.json
   ```
   *(Если настройки дублируются, проверьте также файл `/opt/AgentDVR/XML/config.json`).*

2. Найдите и отредактируйте следующие строки:
   * Нажмите `Ctrl + W`, найдите `TrustedProxies` и пропишите локальный адрес:
     ```json
     "TrustedProxies": "127.0.0.1",
     ```
   * Нажмите `Ctrl + W`, найдите `BlockExternalAccess` и убедитесь, что значение отключено:
     ```json
     "BlockExternalAccess": false,
     ```
3. Сохраните изменения: `Ctrl + O` -> `Enter` -> `Ctrl + X`.

4. Перезапустите службу Agent DVR для применения настроек:
   ```bash
   sudo systemctl restart AgentDVR.service
   ```

## Шаг 3. Установка и настройка Nginx в Ubuntu

Nginx будет принимать внешние запросы на порту `8888`, очищать заголовки с вашим реальным внешним IP и передавать запрос в Agent DVR локально (от имени `127.0.0.1`).

1. Установите веб-сервер Nginx:
   ```bash
   sudo apt update && sudo apt install nginx -y
   ```

2. Откройте конфигурационный файл по умолчанию:
   ```bash
   sudo nano /etc/nginx/sites-available/default
   ```

3. Полностью удалите старое содержимое (`Ctrl + K` для быстрого удаления строк) и вставьте следующий чистый конфигурационный блок:

   ```nginx
   server {
       listen 8888;

       location / {
           proxy_pass http://127.0.0.1:8090;

           # Маскируем хост под локальный
           proxy_set_header Host "127.0.0.1:8090";
           
           # Жёстко затираем внешние IP, передавая вместо них localhost
           proxy_set_header X-Real-IP "127.0.0.1";
           proxy_set_header X-Forwarded-For "127.0.0.1";
           proxy_set_header X-Forwarded-Proto "http";

           # Критически важно для WebSockets (трансляция видеопотока)
           proxy_http_version 1.1;
           proxy_set_header Upgrade \$http_upgrade;
           proxy_set_header Connection "upgrade";
           
           # Отключение буферизации для потокового видео
           proxy_buffering off;
       }
   }
   ```

4. Проверьте конфигурацию на синтаксические ошибки:
   ```bash
   sudo nginx -t
   ```
5. Перезапустите службу Nginx:
   ```bash
   sudo systemctl restart nginx
   ```

---

## Шаг 4. Проверка и использование

1. Для проверки откройте браузер на удаленном устройстве (смартфон или внешний ПК).
2. **Важно:** Первую проверку делайте строго в **режиме инкогнито** (чтобы браузер не отобразил старую закэшированную ошибку блокировки).
3. Перейдите по адресу:
   ```http
   http://YOUR_PUBLIC_IP:8888/
   ```
4. Если интерфейс открылся, но видео не пошло сразу — **кликните мышкой в любом месте экрана**, чтобы дать браузеру разрешение на запуск видеопотока.
